## 二、测试数据解析与断言

### 2.1 JSON / YAML 解析

接口测试中最常见的数据格式是 JSON 和 YAML（配置文件）。

```python
import json

response = '{"code": 0, "data": {"id": 1, "name": "test"}}'
resp_obj = json.loads(response)

# 安全取值，避免 KeyError
assert resp_obj.get("code") == 0
assert resp_obj.get("data", {}).get("name") == "test"
```

```python
import yaml

with open("test_config.yaml", encoding="utf-8") as f:
```

### 2.2 文本与日志解析

```python
import re

log_line = "2026-06-16 10:00:01 ERROR [order-service] payment failed: timeout"
pattern = r"(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) (\w+) \[(.*?)\] (.*)"
match = re.search(pattern, log_line)
if match:
    timestamp, level, service, message = match.groups()
```

**测试开发应用**：
- 从日志中提取错误特征用于失败归因。

- 从 Allure/日志报告中抓取关键指标。

### 2.3 复杂字段断言

```python
from jsonpath_ng import parse

def assert_jsonpath(data, expr, expected):
    values = [m.value for m in parse(expr).find(data)]
    assert values, f"表达式 {expr} 未匹配到数据"
    assert all(v == expected for v in values), f"期望 {expected}，实际 {values}"
```

## 三、测试数据生成

### 3.1 基础随机数据

```python
import random
import string
from datetime import datetime, timedelta

def random_string(length=8):
    return "".join(random.choices(string.ascii_letters + string.digits, k=length))

def random_phone():
    return f"1{random.choice('35789')}{''.join(random.choices(string.digits, k=9))}"

def random_date(start_days=-30, end_days=30):
    return (datetime.now() + timedelta(days=random.randint(start_days, end_days))).strftime("%Y-%m-%d")
```

### 3.2 基于规则批量生成用例数据

```python
def generate_users(count=10):
    return [
        {"username": f"user_{i}", "age": random.randint(18, 60), "role": random.choice(["admin", "guest"])}
        for i in range(count)
    ]
```

### 3.3 数据脱敏

```python
import re

def mask_phone(phone: str) -> str:
    return re.sub(r"(\d{3})\d{4}(\d{4})", r"\1****\2", phone)

def mask_email(email: str) -> str:
    user, domain = email.split("@")
    return f"{user[0]}***@{domain}"
```

## 四、Mock 与 Patch

`unittest.mock` 是 Python 标准库提供的测试替身工具，不依赖任何测试框架。

### 4.1 替换对象行为

```python
from unittest.mock import Mock

api = Mock()
api.get_user.return_value = {"id": 1, "name": "alice"}
assert api.get_user(1)["name"] == "alice"
```

### 4.2 临时替换模块属性

```python
from unittest.mock import patch

def test_send_notification():
    with patch("myapp.notify.send_email") as mock_send:
        mock_send.return_value = True
        result = notify_user("user@example.com")
        assert result is True
        mock_send.assert_called_once_with("user@example.com")
```

### 4.3 模拟异常场景

```python
with patch("requests.get") as mock_get:
    mock_get.side_effect = TimeoutError("connection timeout")
    with pytest.raises(TimeoutError):
        fetch_data("https://api.example.com")
```

## 五、并发执行测试辅助任务

### 5.1 多线程批量接口调用

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

def batch_call(urls, max_workers=4):
    results = {}
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        future_to_url = {executor.submit(requests.get, url): url for url in urls}
        for future in as_completed(future_to_url):
            url = future_to_url[future]
            try:
                results[url] = future.result()
            except Exception as e:
                results[url] = e
    return results
```

## 六、重试机制

```python
from functools import wraps
import time

def retry(exceptions=(Exception,), max_attempts=3, delay=1):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts:
                        raise
                    time.sleep(delay)
        return wrapper
    return decorator

@retry(exceptions=(ConnectionError,), max_attempts=3)
def unstable_api_call():
```

## 七、装饰器在测试工具中的应用

### 7.1 用例执行计时

```python
import time
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} 耗时 {time.time() - start:.3f}s")
        return result
    return wrapper
```

### 7.2 结果缓存

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def get_env_config(env):
    return load_config_from_remote(env)
```

### 7.3 参数校验

```python
def validate_non_empty(*arg_names):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for name in arg_names:
                if not kwargs.get(name):
                    raise ValueError(f"{name} 不能为空")
            return func(*args, **kwargs)
        return wrapper
    return decorator
```

## 八、实践建议

1. **解析逻辑单独封装**：不要把 JSON/正则解析散落在每个用例里，封装为 `utils/parse.py`。
2. **Mock 边界要清晰**：只 mock 外部依赖（HTTP、数据库、消息队列），不要 mock 被测函数内部逻辑。
3. **并发要有度**：线程数不是越大越好，根据目标系统容量设置 `max_workers`。
4. **重试要可观测**：重试装饰器应记录每次失败原因，便于排查 flaky 问题。
5. **数据生成要确定性**：随机数据建议固定随机种子或记录生成参数，保证问题可复现。
