# 常用数据结构

> 来源：`飞书文档/数据结构与算法.md`  
> 整理日期：2026-06-16  
> 适用场景：选择合适的数据结构处理测试数据、评估脚本性能、理解时间/空间复杂度

---

## 一、线性表

### 1.1 数组（Array）

- **特点**：连续内存、随机访问 O(1)、插入/删除 O(n)。
- **测试应用**：存储固定长度的用例参数列表、批量结果收集。

### 1.2 链表（Linked List）

- **特点**：非连续内存、插入/删除 O(1)、查找 O(n)。
- **测试应用**：实现 LRU 缓存、操作历史记录。

| 类型 | 特点 |
|---|---|
| 单链表 | 只能向后遍历 |
| 双向链表 | 可前后遍历，删除/插入更灵活 |
| 循环链表 | 尾节点指向头节点，适合环形数据处理 |

### 1.3 栈（Stack）

- **特点**：后进先出 LIFO。
- **测试应用**：括号匹配检查、表达式求值、调用链回溯、撤销操作。

### 1.4 队列（Queue）

- **特点**：先进先出 FIFO。
- **测试应用**：任务调度、消息队列测试、BFS 遍历。

## 二、散列表（Hash Table）

- **特点**：通过哈希函数将键映射到数组索引，平均查找/插入/删除 O(1)。
- **冲突解决**：开放寻址法、链地址法。
- **负载因子**：元素数/容量，超过阈值需扩容。

**测试应用**：
- 去重失败用例 ID。
- 快速判断元素是否存在于期望集合中。
- 统计日志中各错误码出现次数。

```python
from collections import Counter

error_codes = ["500", "404", "500", "503", "500"]
counter = Counter(error_codes)
print(counter["500"])  # 3
```

## 三、树与图（概述）

### 3.1 二叉树

- **特点**：每个节点最多两个子节点。
- **测试应用**：
  - 二叉搜索树（BST）用于有序数据快速查找。
  - 表达式树用于解析复杂断言条件。

### 3.2 堆（Heap）

- **特点**：优先队列的底层结构，取最值 O(1)，插入/删除 O(log n)。
- **测试应用**：Top-N 慢请求统计、用例优先级调度。

### 3.3 图

- **特点**：由节点和边组成，可表示复杂依赖关系。
- **测试应用**：
  - 测试用例依赖分析。
  - 服务调用链路图。
  - 测试执行顺序拓扑排序。

## 四、复杂度分析

### 4.1 时间复杂度

| 复杂度 | 名称 | 说明 |
|---|---|---|
| O(1) | 常数阶 | 与输入规模无关 |
| O(log n) | 对数阶 | 二分查找 |
| O(n) | 线性阶 | 遍历一次 |
| O(n log n) | 线性对数阶 | 快速排序、归并排序 |
| O(n²) | 平方阶 | 双重循环 |
| O(2ⁿ) | 指数阶 | 组合爆炸，尽量避免 |

### 4.2 空间复杂度

| 复杂度 | 说明 |
|---|---|
| O(1) | 原地操作，额外空间固定 |
| O(n) | 需要额外数组或哈希表 |
| O(n²) | 二维数组、邻接矩阵 |

## 五、常用排序算法速览

| 算法 | 平均时间复杂度 | 空间复杂度 | 稳定性 | 测试应用 |
|---|---|---|---|---|
| 冒泡排序 | O(n²) | O(1) | 稳定 | 教学，不推荐生产使用 |
| 插入排序 | O(n²) | O(1) | 稳定 | 小规模数据 |
| 选择排序 | O(n²) | O(1) | 不稳定 | 教学 |
| 快速排序 | O(n log n) | O(log n) | 不稳定 | 通用排序 |
| 归并排序 | O(n log n) | O(n) | 稳定 | 链表排序、外部排序 |
| 堆排序 | O(n log n) | O(1) | 不稳定 | Top-K 问题 |

## 六、实践建议

1. **优先使用内置数据结构**：Python 的 `list`、`dict`、`set`、`collections.deque` 已高度优化。
2. **去重首选 `set`**：比列表遍历去重效率更高。
3. **计数首选 `Counter`**：避免手写统计逻辑。
4. **大数据量避免 O(n²)**：双重循环处理百万级数据会超时。
5. **理解递归的栈开销**：深度递归可能导致栈溢出，可改用迭代或增加递归限制。
