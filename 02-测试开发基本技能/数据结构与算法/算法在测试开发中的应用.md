## 一、日志解析与模式匹配

测试开发中常需要从非结构化日志中提取关键信息。

### 1.1 正则匹配

```python
import re

log = "2026-06-16 10:00:01 ERROR [order-service] payment timeout: order_id=12345"
pattern = r"(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) (\w+) \[(.*?)\] (.*?) order_id=(\d+)"
match = re.search(pattern, log)
if match:
    timestamp, level, service, message, order_id = match.groups()
```

### 1.2 多行日志合并

```python
from itertools import groupby

def group_multiline_logs(lines):
    """按时间戳前缀合并多行日志"""
    groups = []
    current = []
    for line in lines:
        if re.match(r"^\d{4}-", line):
            if current:
                groups.append("".join(current))
            current = [line]
        else:
            current.append(line)
    if current:
        groups.append("".join(current))
    return groups
```

### 2.3 分层采样

按业务模块/优先级分层抽样，保证覆盖度。

```python
from collections import defaultdict

def stratified_sample(cases, by="module", n_per_group=10):
    groups = defaultdict(list)
    for case in cases:
        groups[case[by]].append(case)
    
    sampled = []
    for group, items in groups.items():
        sampled.extend(random.sample(items, min(n_per_group, len(items))))
    return sampled
```

## 四、图与依赖关系分析

### 4.1 拓扑排序确定执行顺序

当用例之间存在依赖时，可用拓扑排序确定执行顺序。

```python
from collections import defaultdict, deque

def topological_sort(tasks, dependencies):
    """
    tasks: 任务列表
    dependencies: {task: [prerequisite_tasks]}
    """
    in_degree = {t: 0 for t in tasks}
    graph = defaultdict(list)
    
    for task, deps in dependencies.items():
        for dep in deps:
            graph[dep].append(task)
            in_degree[task] += 1
    
    queue = deque([t for t in tasks if in_degree[t] == 0])
    result = []
    
    while queue:
        node = queue.popleft()
        result.append(node)
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    if len(result) != len(tasks):
        raise ValueError("存在循环依赖")
    return result
```

### 4.2 服务调用链路分析

```python
# 基于依赖图找出影响范围
def find_affected_services(graph, changed_service):
    """从变更服务出发，BFS 找出所有受影响的服务"""
    visited = {changed_service}
    queue = deque([changed_service])
    while queue:
        node = queue.popleft()
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return visited
```

## 五、二分查找

在有序数据中快速定位，时间复杂度 O(log n)。

```python
def binary_search(sorted_list, target):
    left, right = 0, len(sorted_list) - 1
    while left <= right:
        mid = (left + right) // 2
        if sorted_list[mid] == target:
            return mid
        elif sorted_list[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

**测试应用**：
- 在有序时间戳列表中查找某次失败附近的日志。
- 在已排序的用例 ID 中快速判断是否存在。

## 六、滑动窗口

用于处理连续子序列问题，如最近 N 条请求、固定时间窗口内的错误率。

```python
from collections import deque

def error_rate_in_window(logs, window_seconds=60):
    """计算最近 window_seconds 秒内的错误率"""
    window = deque()
    total = error = 0
    now = logs[-1]["timestamp"]
    
    for log in reversed(logs):
        if now - log["timestamp"] > window_seconds:
            break
        total += 1
        if log["level"] == "ERROR":
            error += 1
    
    return error / total if total else 0
```

## 七、字符串匹配与 diff

### 7.1 最长公共子序列（LCS）

用于比较两个文本/输出的相似度。

```python
def lcs_length(a, b):
    m, n = len(a), len(b)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if a[i-1] == b[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```

### 7.2 使用 difflib

```python
import difflib

def text_diff(expected, actual):
    diff = difflib.unified_diff(
        expected.splitlines(),
        actual.splitlines(),
        lineterm=""
    )
    return "\n".join(diff)
```

## 八、实践建议

1. **不要为简单场景硬套复杂算法**：列表去重用 `set`，排序用内置 `sorted`。
2. **关注数据规模**：百万级以上才需要考虑算法优化。
3. **优先使用标准库**：`itertools`、`collections`、`bisect`、`heapq`、`difflib` 已覆盖大部分需求。
4. **图算法用于依赖分析**：用例依赖、服务依赖、模块影响面分析都可用图建模。
5. **复杂度意识**：写脚本前先估算数据量级，避免 O(n²) 处理大数据。
