# Claude Code 核心概念与应用

> 来源：`飞书文档/Claude Code 核心概念总结.md`、`飞书文档/内置工具总览.md`  
> 整理日期：2026-06-16  > 适用场景：使用 Claude Code 辅助测试开发、编写用例、排查失败、重构脚本

---

## 一、Claude Code 是什么

Claude Code 是运行在终端的 AI 编程助手。它通过**智能体循环（Agentic Loop）**自主完成任务：收集上下文 → 执行操作 → 验证结果。用户可随时打断并调整方向。

---

## 二、核心工作循环

```text
收集上下文 → 执行操作 → 验证结果
   ↑___________________________|
```

1. **收集上下文**：自动搜索文件、理解代码结构、读取相关文档。
2. **执行操作**：编辑文件、运行命令、调用工具。
3. **验证结果**：运行测试、检查输出、确认修改。

---

## 三、Claude Code 能访问什么

在某一目录运行 `claude` 后，它可以访问：

- **项目文件**：当前目录及子目录中的文件。
- **终端**：可执行命令行工具和脚本。
- **Git 状态**：当前分支、未提交更改、近期提交历史。
- **CLAUDE.md**：项目级持久化指令文件。
- **自动记忆**：会话中积累的项目知识。
- **扩展**：MCP 服务器、Skills、Subagents。

---

## 四、内置工具一览

Claude Code 通过调用内置工具与代码库交互。测试开发中高频工具：

| 工具 | 作用 | 测试开发场景 |
|---|---|---|
| `Read` | 读取文件 | 查看用例代码、配置文件 |
| `Write` | 创建/覆盖文件 | 生成新测试脚本 |
| `Edit` | 精准修改文件 | 修复断言、调整用例 |
| `Bash` | 执行命令 | 运行 pytest、查看日志 |
| `Grep` | 搜索文件内容 | 查找失败用例、定位代码 |
| `Glob` | 按模式匹配文件 | 批量发现测试文件 |
| `Agent` | 生成子代理 | 委派独立任务 |
| `AskUserQuestion` | 向用户提问 | 需要确认时 |
| `TaskCreate/TaskUpdate` | 任务管理 | 复杂多步骤任务 |

## 五、测试开发高频应用场景

### 5.1 快速生成测试用例

```text
提示词：
"为 user_login 接口生成 5 条异常场景用例，覆盖：
- 用户名不存在
- 密码错误
- 密码为空
- 账号被锁定
- 请求频率超限

使用 pytest + requests，断言返回状态码和错误消息。"
```

### 5.2 失败日志分析

```text
提示词：
"这是 pytest 的失败日志，请分析根因并给出修复建议：
[粘贴日志]"
```

### 5.3 重构测试代码

```text
提示词：
"将 tests/legacy/ 目录下的重复 setup 代码提取为 conftest.py 中的 fixture，
保持现有用例行为不变。"
```

### 5.4 批量修改

```text
提示词：
"把所有测试文件中的 assert response.status_code == 200
替换为统一的 assert_response_ok(response) 辅助函数。"
```

## 六、高效使用技巧

1. **像对话一样工作**：不需要完美提示词，直接表达需求，迭代修正。
2. **前期说清楚**：初始提示越精确，后续修正越少。引用具体文件、说明约束条件。
3. **给可验证的目标**：提供测试用例、预期输出或截图，Claude 能自我验证时效果最好。
4. **委托而非指挥**：给出背景和方向，让 Claude 决定读哪些文件、运行什么命令。
5. **复杂任务先用 Plan Mode**：按 `Shift+Tab` 两次进入计划模式，先分析再执行。
6. **保持上下文聚焦**：长时间会话后，主动总结关键决策，避免上下文漂移。

## 七、扩展机制

| 扩展 | 作用 | 测试开发应用 |
|---|---|---|
| **Skills** | 封装可复用的工作流 | 定义"生成接口用例""分析失败日志"等技能 |
| **MCP** | 连接外部服务 | 连接数据库、Jira、测试平台获取上下文 |
| **Hooks** | 自动化工作流 | 提交前自动跑 lint、提交后触发用例生成 |
| **Subagents** | 委派子任务 | 让子代理专门处理日志分析或数据构造 |

## 八、.claude 目录配置

Claude Code 从项目目录的 `.claude/` 和用户主目录的 `~/.claude/` 读取配置：

```text
.claude/
├── CLAUDE.md          # 项目级指令
├── skills/            # 自定义技能
├── hooks/             # 自动化钩子
└── memory/            # 自动记忆
```

---

## 九、实践建议

1. **为测试项目编写 CLAUDE.md**：说明测试框架、命名规范、运行命令，减少每次重复说明。
2. **用 Skills 固化高频任务**：把"生成 Pytest 用例模板""分析 Allure 报告"等变成可调用技能。
3. **AI 生成代码必须人工审查**：不要直接合并 AI 生成的代码，尤其是断言和边界条件。
4. **敏感数据不上传**：避免把生产数据、密码、密钥放入 AI 上下文。
5. **保留关键决策记录**：AI 建议的方案要落回文档，避免下次重复推理。
