# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库性质

这是一个 **Obsidian 本地知识库**，不是软件工程代码库。内容围绕测试开发专业领域沉淀，以 Markdown 文件为主体，配合 YAML frontmatter、双向链接和模板工作流。

## 常用命令

本仓库没有构建、测试或 lint 脚本。日常维护主要使用 Git 和标准 shell 工具：

- 查看当前变更：`git status`
- 查看工作区差异：`git diff`
- 新增/修改文档后自行决定提交时机（用户要求自己掌控提交时机，不要主动提交）。
- 检查是否有孤立或未链接的文档：搜索未出现在 `MOC-首页.md` 或相关 MOC 中的文件。
- 批量替换链接或调整目录结构后，使用 Obsidian 的「更新反向链接」功能，或在 `.obsidian/app.json` 中确认 `alwaysUpdateLinks: true`。

## 知识库架构

### 目录组织

采用 **"一级领域 → 二级模块 → 三级主题"** 的层级，但仅通过目录 + MOC 管理顺序，正文标题不再嵌入编号：

- `00-inbox/`：临时收集箱，碎片想法、待整理素材先放这里。
- `01-测试基础与方法论/` 到 `08-AI与智能化测试/`：8 个专业领域目录。
  - `01` 为"道"（测试思维、质量观）。
  - `02` 为"器"（通用技术栈：Python、Linux、网络、Git 等）。
  - `03` 自动化测试工程、`04` 云计算领域测试、`08` AI 与智能化测试 为三大核心支柱。
  - `05` 专项测试技术、`06` 质量效能平台与数据、`07` 团队组织力与影响力 为扩展与组织级能力。
- `_archive/`：归档区。
- `_templates/`：文档模板库。
- `assets/`：图片、截图、PDF 等附件（在 `.obsidian/app.json` 中配置为默认附件目录）。
- `MOC-首页.md`：知识库唯一总入口，承载学习地图、建设进度和跨域关联。
- `README.md`：仓库规范、命名规则、标签体系、模板选择指南。
- `知识库设计方案.md`：设计背景、目录规划、迁移映射、实施路线。

### 命名规范

- 目录名：`[编号]-主题名`，如 `03-自动化测试工程`、`08-AI与智能化测试`。
- 文件名：`主题名.md`，不嵌入编号，保持简洁。
- 页面标题：与文件名保持一致，不嵌入编号。
- 图片命名：`领域-主题-序号.png`。

### 文档元数据

每篇文档顶部使用 YAML frontmatter：

```yaml
---
title: "文档标题"
tags:
  - 文档类型标签
  - 技术栈或场景标签
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
domain: "所属一级领域"
project: "项目名称（项目文档必填）"
---
```

常用文档类型标签：`理论`、`实战`、`项目`、`复盘`、`规范`、`速查`。

### 模板选择

新建文档时，根据内容类型从 `_templates/` 选择模板：

- `通用知识笔记模板.md`：方法论、原则、最佳实践、能力模型。
- `项目实战文档模板.md`：具体业务项目的测试实践、简历素材沉淀，必须回答简历四问。
- `速查表模板.md`：命令、配置、参数、快捷参考。
- `工具使用文档模板.md`：具体工具的安装、配置和使用。
- `技术方案设计文档模板.md`：技术方案、架构设计类内容。

### 链接与入口维护

- 使用 `[[文件名]]` 在文档间建立双向链接。
- 新增或调整重要文档后，回到 `MOC-首页.md` 更新入口和状态。
- 避免产生"幽灵节点"：不要在 MOC 或文档中引用不存在的文件或已被移动的目录。

### AI 域特殊设计

`08-AI与智能化测试` 是独立知识域，按"AI 基础能力 → AI 工程化落地 → AI 辅助测试开发 → AI 应用测试 → AI 安全测试 → 多模态与工程化平台"组织。AI 知识更新快，按三层节奏沉淀：

- 趋势层：每月，只记一句话结论和链接。
- 方法层：每季度，沉淀可迁移的方法论。
- 资产层：每半年，形成可复用的模板/脚本。

## 内容质量标准

- 文档正文开头需写 1-2 句话摘要，说明解决什么问题。
- 项目实战文档必须回答简历四问：做了什么、解决什么业务问题、用了什么技术和方法、取得什么可量化结果。
- 知识按领域聚合，不按来源拆分；通用知识和项目经验放在同一目录下，通过 `tags` 区分类型。
