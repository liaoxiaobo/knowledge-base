# 提示词工程

## 一句话本质

提示词工程是通过设计、优化和约束输入文本，引导大语言模型稳定输出高质量结果的工程化方法。

## 核心内容

1. **Prompt 的基本结构**：角色（Role）+ 上下文（Context）+ 任务（Task）+ 输出要求（Output Format）+ 约束（Constraints）。
2. **零样本与少样本提示**：零样本依赖模型先验知识；少样本通过示例引导模型对齐输出格式与风格。
3. **思维链（Chain-of-Thought, CoT）**：在 Prompt 中要求模型“一步步思考”，可显著提升复杂推理任务的正确率。
4. **Prompt 模板化**：将可复用的提示词固化为模板，配合变量注入，实现批量、稳定的生成任务。
5. **测试场景中的 Prompt 设计**：测试用例生成、缺陷描述润色、失败日志分析、接口测试断言生成等。
6. **Prompt 评估与迭代**：建立评估维度（正确性、一致性、完整性、安全性），通过 A/B 对比持续优化。
7. **提示词注入与安全**：防范用户输入覆盖系统指令、泄露敏感信息或诱导模型输出违规内容。

## 适用场景

- 使用 LLM 批量生成测试用例或测试数据。
- 让 AI 编程工具（如 Claude Code）按统一规范输出代码、注释和提交信息。
- 构建 AI 辅助失败归因、日志摘要、根因定位等内部工具。
- 沉淀团队级 Prompt 模板库，降低 AI 工具使用门槛。

## 典型反模式

- 把所有上下文一次性塞进 Prompt，导致关键信息被截断或稀释。
- 不给输出格式约束，导致每次输出结构不一致，难以解析。
- 只依赖单次生成结果，不做评估和迭代。
- 在 Prompt 中暴露敏感业务数据或内部系统架构。

## 最小可复现实践

1. 为常用测试任务写一个“角色+任务+格式+约束”的 Prompt 模板。
2. 收集 5-10 个真实输入，分别用两个版本 Prompt 生成输出并对比。
3. 把效果最好的版本按场景整理到本目录，作为团队 Prompt 模板库，标注适用场景和注意事项。
