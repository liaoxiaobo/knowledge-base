# AI 测试开发工程师学习路线图

> 本文基于 2026 年 AI 技术发展趋势与测试开发岗位招聘要求整理，面向希望系统性学习 AI、并将 AI 能力落地到测试开发工作中的工程师。

---

## 一、2026 年外部趋势摘要

### 1.1 AI 技术发展趋势

2026 年 AI 发展的主线是 **大模型 Agent 化、实战化、企业化**：

1. **从预训练转向后训练与 RLVR**：技术重心从预训练规模竞赛，转向 RLHF/RLVR/DPO 等后训练阶段，目标是让模型成为能在真实业务场景中解决复杂问题的"实干型 AI 员工"。
2. **Agent 成为核心突破口**：大模型从聊天工具进化为具备推理、规划、执行多步任务能力的自主 Agent，应用形态下沉到本地设备、操作系统和企业业务流程。
3. **MCP 成为 AI 集成基础设施**：MCP（Model Context Protocol）正在快速成为 AI Agent 连接工具与数据源的统一标准，类比"AI 的 USB-C 接口"。
4. **多 Agent 协同与企业级安全**：分层 Agent 系统、人在环审批、细粒度授权、审计日志成为企业落地的关键。
5. **多模态扩展**：从文本扩展到视频、音频、视觉（VLM）等多模态能力。

### 1.2 测试开发招聘要求

2026 年"测试开发工程师（AI 大模型方向）"岗位已从传统软件测试转向 **"测试工程能力 + AI/LLM 领域知识 + 自动化/评测体系建设"** 的复合型人才：

- **工程基础**：Python/Java、Linux、Docker/K8s、CI/CD、数据库、微服务。
- **测试能力**：功能/接口/自动化/性能/稳定性测试，Pytest/JUnit 等框架。
- **AI 领域知识**：
  - LLM 原理：Transformer、Attention、预训练-微调、SFT、LoRA、RLHF/DPO。
  - 应用范式：Prompt 工程、RAG、Agent、MCP、LangChain、Dify。
  - 评测能力：BLEU/ROUGE、幻觉率、准确率、鲁棒性、红队测试、Benchmark 设计。
- **加分项**：模型训练/推理/部署经验、多模态（VLM/YOLO）、行业知识（汽车/金融/医疗）。

### 1.3 AI 测试工程师核心技能

| 能力域 | 关键内容 |
| --- | --- |
| Prompt Engineering | 结构化、角色化、Few-Shot、Chain-of-Thought 等技巧 |
| LLM 测试与评估 | 幻觉率、响应延迟、输出稳定性、监控告警 |
| RAG 测试 | 检索质量、生成质量、知识库测试、端到端评估 |
| Agent 测试 | 工具调用、任务规划、异常处理、边界与安全 |
| 多模态与垂直场景 | VLM、车载/金融/医疗等行业应用测试 |
| 软技能 | 业务洞察、风险策略、伦理安全治理、批判性思维 |

## 二、06 域知识库结构

本知识库 06 域按 **"AI 基础能力 → AI 工程化落地 → AI 辅助测试开发 → AI 应用测试 → AI 安全测试 → 多模态与工程化平台"** 组织：

```
06 AI 与智能化测试
├── 6.1 AI测试通识
├── 6.2 提示词与Agent编排
├── 6.3 RAG与知识代理
├── 6.4 Agent与Skill
├── 6.5 AI编程工具
├── 6.6 AI辅助测试开发
├── 6.7 AI应用测试
├── 6.8 AI安全与对齐测试
├── 6.9 多模态与垂直场景测试
└── 6.10 AI测试工程化与评测平台
```

## 三、推荐学习路径

### 路径一：快速上手型（适合想马上提效）

**目标**：最快把 AI 用到日常测试开发工作中。

| 顺序  | 子域                                     | 为什么先学                                         |
| --- | -------------------------------------- | --------------------------------------------- |
| 1   | **6.5 AI编程工具**                         | 立竿见影，Claude Code、Cursor 等工具能立刻帮你写脚本、改 Bug、看代码 |
| 2   | **6.2 提示词与工具调用**                       | 工具用得好不好，取决于 Prompt 能力，这是"新代码能力"               |
| 3   | **6.6 AI辅助测试开发**                       | 把 AI 用到用例生成、脚本生成、接口/UI 测试、失败归因中               |
| 4   | **6.1 AI测试通识**                         | 补 LLM 基础机制、评估指标、推理测试基础，建立系统认知                 |
| 5   | **6.3 RAG与知识代理** + **6.4 Agent与Skill** | 理解企业 AI 应用的主流架构：MCP、Agent、RAG                 |
| 6   | **6.7 AI应用测试** + **6.8 AI安全与对齐测试**     | 从"用 AI 做测试"转向"测试 AI"                          |
| 7   | **6.10 AI测试工程化与评测平台**                  | 把单点能力沉淀为评测流水线、数据集管理、CI/CD 门禁                  |
| 8   | **6.9 多模态与垂直场景测试**                     | 进阶方向，根据业务需要选学                                 |

### 路径二：系统构建型（适合打牢基础）

**目标**：建立完整的 AI 测试知识体系，向 AI 测试架构师发展。

| 顺序 | 子域 | 学习重点 |
| --- | --- | --- |
| 1 | **6.1 AI测试通识** | 建立基础：LLM 原理、评估指标、推理服务测试 |
| 2 | **6.2 提示词与工具调用** | 掌握与 LLM 交互的核心技能 |
| 3 | **6.5 AI编程工具** | 用 Claude Code 等工具提升工程效率 |
| 4 | **6.3 RAG与知识代理** | 理解检索增强生成和企业知识库测试 |
| 5 | **6.4 Agent与Skill** | 理解 Agent 架构、工作流、MCP、Skill 设计 |
| 6 | **6.6 AI辅助测试开发** | 将 AI 能力落地到测试设计、脚本生成、维护、失败分析 |
| 7 | **6.7 AI应用测试** | 系统学习 LLM/Agent/RAG 应用怎么测 |
| 8 | **6.8 AI安全与对齐测试** | 学习提示词攻击、红队测试、偏见隐私、合规 |
| 9 | **6.10 AI测试工程化与评测平台** | 搭建可持续的 LLM 评测体系和 CI/CD 门禁 |
| 10 | **6.9 多模态与垂直场景测试** | 扩展到 VLM、多模态和行业垂直场景 |

## 四、按时间投入的学习方案

### 如果你只有 1 个月

建议只学这 4 个，学完就能在简历上写"具备 AI 辅助测试开发能力"：

1. **6.5 AI编程工具** — 每天用它工作
2. **6.2 提示词与工具调用** — 把 Prompt 写稳
3. **6.6 AI辅助测试开发** — 至少落地一个 AI 生成脚本或用例的项目
4. **6.1 AI测试通识** — 面试和方案设计时够用

### 如果你有 3 个月

在上面的基础上，增加：

5. **6.3 RAG与知识代理** — 理解企业最常见的 LLM 落地架构
6. **6.4 Agent与Skill** — 理解 Agent 系统和 MCP
7. **6.7 AI应用测试** — 开始建立"测试 AI"的能力

### 如果你有 6 个月以上

系统走完 6.1-6.10，重点突破：

8. **6.8 AI安全与对齐测试** — 红队测试、对抗样本、合规治理
9. **6.10 AI测试工程化与评测平台** — 评测流水线、数据集管理、可观测、CI/CD
10. **6.9 多模态与垂直场景测试** — 根据所在行业选学

## 五、关键学习原则

1. **6.5 是最快的入口**：不管哪条路径，都建议先花 1-2 周把 Claude Code 用到顺手。
2. **6.1 是地基**：想理解 AI 为什么出错、怎么评估，必须回来看 6.1。
3. **6.6 是本职**：测试开发工程师的核心差异化能力，是把 AI 用到测试活动中。
4. **6.7 / 6.8 是进阶壁垒**：能"测试 AI"的人比"会用 AI 做测试"的人更稀缺。
5. **6.10 是平台化能力**：适合有 1-2 年 AI 测试经验后，向架构师方向发展时重点突破。
